## 3. BM1366 Protocol — Applicable to BM2042AA (High Confidence)

Skot (BitAxe creator) has fully documented the BM1366 protocol at `skot/BM1397/bm1366_protocol.md`. The BM2042AA almost certainly uses the same framing since all Bitmain ASICs follow the same UART convention.

### UART parameters (BM1366, expected same for BM2042AA)
```
Baud rate: 115200 (default), configurable higher
Data bits: 8
Parity: None
Stop bits: 1
(115200 8N1)
```

### Pin functions (BM1366, expect same)
- **CI** — Command Input (UART RX into chip)
- **RO** — Response Output (UART TX from chip)  
- **CLKI** — 25MHz clock input (propagates to CLKO for daisy-chain)
- **CLKO** — Clock output to next chip in chain
- **NRSTI** — Active-low reset
- **BI** — Busy Input — must be pulled LOW to enable communication

### Command frame format (confirmed for BM1366)
```
Preamble: 0x55 0xAA (2 bytes, fixed)
TYPE:  1 = send Job | 2 = send Command
ALL:   0 = single chip | 1 = all chips in chain
CMD:   varies by TYPE (see below)
LEN:   frame length excl. preamble
DATA:  job or register data
CRC:   CRC5 or CRC16 depending on command type
```

### Response frame (confirmed for BM1366)
```
Preamble: 0xAA 0x55 (note: reversed from command)
Length: 9 bytes (fixed)
TYPE:  0 = register response | 4 = nonce found
```

### Job sending (TYPE=1, CMD=1)
Data field contains: midstate + block header tail  
For CKB: will need full ~200-byte block header (vs Bitcoin's 80 bytes)  
Nonce field: 64-bit for CKB (vs 32-bit for Bitcoin)

### Key firmware difference for Eaglesong
The BM2042AA job format will differ from BM1366 in the DATA field:
- Different header structure (CKB block header)
- Larger nonce space (64-bit)
- Eaglesong midstate (if the chip supports midstate optimisation — unknown)

## 4. What Skot's Work Gives Us For Free

The `skot/BM1397` and `skot/ESP-Miner` repos have done the hard protocol work for SHA256 ASICs. Our firmware fork path:

1. Keep all UART framing code (`0x55 0xAA` preamble, CRC5/CRC16, address chain init)
2. Replace the `send_job` DATA field construction with CKB block header format
3. Replace nonce parsing with 64-bit nonce handling
4. Replace stratum task with CKB stratum (already done in NerdMiner_CKB)
5. Adjust baud rate / clock init if BM2042AA differs from BM1366 defaults
